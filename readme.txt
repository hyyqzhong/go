dpos共识算法：
启动：main函数，

名词描述：
候选人（节点）：包括受托人和非受托人，投票结果选出来后的就是受托人
本地数据桶库：基于Go的K/V数据库，github中的
前言：
1.本应用采用本地数据桶库，将交易数据、候选人记录（包括投票数）、区块等均存储至数据库；
2.本地产生交易后将通过tcp（传输控制协议）将数据同步至其他节点；
3.当某个节点启动tcp连接时，将该节点通过tcp协议点播至其他节点，其他节点获取到后存库；
4.投票结果后会将是否受托人标识通过tcp协议点播其他节点，其他节点获取到后才有权限进行出块（delegateFlag标识）；
5.目前出块会涉及merkel树根计算，然后再计算hash工作量，目前是3个0输出；
6.每次产生新块后会通过tcp协议广播至其他节点，其他节点收到新块通过一个blackFlag标识进行停止自身出块计算；并开始进行接收到区块的验证；
验证通过存储至本地库，并广播出去；
